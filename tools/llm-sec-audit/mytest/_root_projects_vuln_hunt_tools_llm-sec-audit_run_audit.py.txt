### 问题 1

- **类型**：路径遍历  
- **位置**：第11行：`code_path = pathlib.Path(args.code_file).resolve()`  
- **严重度**：高  
- **描述**：用户通过 `--code-file` 参数传入的文件路径未经任何限制，程序直接使用 `pathlib.Path.resolve()` 解析后读取文件内容（第15行），并将内容传递给 LLM 子进程。攻击者可以读取系统上任意文件（如 `/etc/shadow`、私钥等），导致敏感信息泄露。  
- **攻击向量**：执行脚本时传入 `--code-file /etc/shadow`，程序读取 `/etc/shadow` 内容并发送给 LLM。  
- **修复建议**：对输入路径进行白名单校验，仅允许读取指定目录下的文件。例如，使用 `os.path.realpath` 并检查是否以允许的基路径开头：  
  ```python
  ALLOWED_DIR = pathlib.Path("/allowed/projects").resolve()
  code_path = pathlib.Path(args.code_file).resolve()
  if not str(code_path).startswith(str(ALLOWED_DIR)):
      sys.exit("Error: code-file must be under allowed directory")
  ```

### 问题 2

- **类型**：路径遍历  
- **位置**：第12行：`prompt_path = pathlib.Path(args.prompt_file).resolve()`  
- **严重度**：高  
- **描述**：用户通过 `--prompt-file` 参数传入的文件路径同样未经限制，该路径被直接传递给子进程（第27、31行）作为 `--prompt-file` 参数，子进程会读取该文件内容。攻击者可读取任意文件，造成信息泄露。  
- **攻击向量**：传入 `--prompt-file /etc/passwd`，子进程读取 `/etc/passwd` 内容。  
- **修复建议**：同问题1，对 `prompt_file` 路径进行白名单校验。

### 问题 3

- **类型**：路径遍历  
- **位置**：第21行：`out = pathlib.Path(args.output_dir) / f"{safe_name}.txt"`  
- **严重度**：中  
- **描述**：用户通过 `--output-dir` 参数控制输出目录，程序将结果写入该目录下的文件（文件名由 `code_path` 经替换斜杠后生成）。攻击者可以指定任意目录（如 `/tmp/evil`、`/etc/cron.d/`），虽然文件名不可控（不含斜杠），但仍可在任意位置创建文件，可能干扰系统或配合其他漏洞利用。  
- **攻击向量**：设置 `--output-dir /etc/cron.d/`，并构造 `code-file` 使 `safe_name` 为合法文件名（如 `_etc_cron.d_evil`），则会在 `/etc/cron.d/` 下创建文件（但文件名含下划线，实际被 cron 执行的可能性低）。更可能写入 web 目录或日志目录。  
- **修复建议**：限制输出目录为固定路径，或对 `output_dir` 进行白名单校验，确保在预期范围内：  
  ```python
  ALLOWED_OUTPUT = pathlib.Path("/allowed/output").resolve()
  output_dir = pathlib.Path(args.output_dir).resolve()
  if not str(output_dir).startswith(str(ALLOWED_OUTPUT)):
      sys.exit("Error: output-dir must be under allowed directory")
  ```

# 完整性检查清单

在结束函数的微观分析之前，验证以下内容：

---

## 结构完整性
- [ ] 用途部分：2+ 句话说明函数角色
- [ ] 输入与假设部分：所有参数 + 隐式输入已记录
- [ ] 输出与影响部分：所有返回值、状态写入、外部调用、事件
- [ ] 逐块分析：每个逻辑块都已分析（无遗漏）
- [ ] 跨函数依赖：所有调用和共享状态已记录

---

## 内容深度
- [ ] 识别了至少 3 个不变量（必须始终成立的条件）
- [ ] 记录了至少 5 个假设（被假定为真的条件）
- [ ] 至少应用了 1 次第一性原理
- [ ] 至少应用了 3 次 5 Whys 或 5 Hows（合计）
- [ ] 对所有外部交互进行了风险分析（重入、恶意合约等）

---

## 连续性与集成
- [ ] 与相关函数交叉引用（如有内部调用，分析被调用函数）
- [ ] 从调用者处传播了假设（如此函数被其他函数调用）
- [ ] 识别了不变量耦合（此函数的不变量如何与全局系统关联）
- [ ] 跨函数边界追踪了数据流（如适用）

---

## 反幻觉验证
- [ ] 所有断言引用了特定行号（L45、第 98-102 行等）
- [ ] 无模糊表述（"可能"、"大概"、"似乎"）——替换为"不明确；需要检查 X"
- [ ] 矛盾已解决（如先前分析与当前发现冲突，已显式更新）
- [ ] 基于证据：每个不变量/假设都与实际代码关联

---

## 完成信号

分析完成的条件：
1. 以上所有检查项均已满足
2. 没有剩余的"TODO: 分析 X"或"不明确的 Y"项
3. 完整调用链已分析（对内部调用，已跳入并分析）
4. 所有已识别风险或有缓解分析，或已确认为未解决

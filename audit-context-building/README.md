# Audit Context Building（审计上下文构建）

在漏洞挖掘之前，通过超细粒度代码分析构建深层架构上下文。

**作者：** Omar Inuya（Trail of Bits）

## 何时使用

在以下场景使用本技能：
- 在安全审计之前需要深度理解代码库
- 构建自底向上的理解，而非高层猜测
- 减少复杂分析过程中的幻觉和上下文丢失
- 为威胁建模或架构评审做准备

## 功能说明

本技能定义了 Claude 在审计上下文构建阶段的思维方式。启用时，Claude 将会：

- 执行**逐行/逐块**代码分析
- 在微观尺度应用**第一性原理（First Principles）**、**5 Whys** 和 **5 Hows**
- 构建并维护稳定、显式的思维模型
- 识别不变量、假设、数据流和推理风险
- 追踪跨函数和外部调用流，并完整传播上下文

## 关键原则

这是一个**纯上下文构建**技能。它**不**：
- 识别漏洞
- 提出修复方案
- 生成漏洞利用证明
- 分配严重性或影响等级

它仅用于在漏洞挖掘阶段之前构建深度理解。

## 安装

```
/plugin install trailofbits/skills/plugins/audit-context-building
```

## 阶段

1. **初始定位** - 映射模块、入口点、参与者和存储
2. **超细粒度函数分析** - 逐行语义分析，跨函数流追踪
3. **全局系统理解** - 状态/不变量重建、工作流映射、信任边界

## 反幻觉规则

- 绝不让证据迁就早期假设
- 当被矛盾证据推翻时，显式更新模型
- 避免模糊猜测；使用"不明确；需要检查 X"
- 持续交叉引用以维护全局一致性

## 相关技能

- `issue-writer` - 在上下文构建完成后撰写发现报告
- `differential-review` - 使用上下文构建进行基线分析
- `spec-compliance` - 将理解的行为与文档进行比较

# 完整性检查清单

在结束暴露面映射之前，逐项验证以下内容。

## 结构完整性
- [ ] 项目概览已包含：语言、框架、模块划分、入口模式、鉴权框架
- [ ] 每个暴露面包含：分类、入口描述、位置、Handler、判断依据
- [ ] 可疑点已单独汇总
- [ ] 分析建议已给出

## 覆盖完整性
- [ ] HTTP 路由已扫描（大多数项目都有，不应遗漏）
- [ ] 如项目使用消息队列，MQ 分类已覆盖
- [ ] 如项目有命令行工具，CLI 分类已覆盖
- [ ] 如项目有定时任务，CRON 分类已覆盖
- [ ] 如项目有文件上传/导入/导出，FILE_IO 已覆盖
- [ ] 如项目有外部 API 调用且 URL 可变，OUTBOUND_HTTP 已覆盖
- [ ] 如项目有 gRPC/Thrift/Dubbo，RPC 分类已覆盖
- [ ] 覆盖了框架支持的所有入口注册方式（不遗漏注解/配置/代码注册）

## 边界控制
- [ ] 内部模块间 RPC 已被排除
- [ ] 内部 EventBus 已被排除
- [ ] 内部 filter/interceptor/middleware 已被排除
- [ ] 被 CLI/Cron handler 调用的业务函数已被排除
- [ ] 固定路径的文件写入已被排除
- [ ] 固定 URL 的健康检查/监控上报已被排除
- [ ] 不确定是否外部可达的条目已标注"不明确"

## 质量验证
- [ ] 每条暴露面引用了具体文件:行号
- [ ] 无模糊表述（"可能"、"大概"、"似乎"）
- [ ] 矛盾已解决（如同一 handler 被多个 route 引用已合并）
- [ ] 未读的文件/目录已如实汇报（不假装扫过）

## 完成信号

暴露面映射完成的条件：
1. 以上所有检查项均已满足
2. 没有剩余的"TODO: 分析 X"项
3. 所有已发现可疑点已汇总
4. 分析建议已被给出

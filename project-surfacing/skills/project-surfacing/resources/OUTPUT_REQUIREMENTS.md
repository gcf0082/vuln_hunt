# 输出要求

执行暴露面映射时，必须按以下格式组织输出。

## 必需结构

### 项目概览
- 语言
- 框架
- 模块划分（简要）
- 入口注册模式
- 鉴权框架

### 暴露面列表

每个暴露面包含以下字段：

| 字段 | 必填 | 说明 |
|---|---|---|
| 分类 | 是 | HTTP / RPC / MQ / CLI / CRON / FILE_IO / OUTBOUND_HTTP |
| 入口描述 | 是 | URL pattern / queue/topic / 命令名 / cron 表达式 / 路径参数 / URL 构造 |
| 位置 | 是 | 文件:行号 |
| Handler | 是 | 函数签名或处理器描述 |
| 鉴权 | 否 | unauthenticated / authenticated / admin / 不明确 |
| 发现 | 否 | 值得注意的观察（无鉴权、参数直接传路径等） |
| 判断依据 | 是 | 为什么认为这是外部可达的 |

### 可疑点列表

- 文件:行号 — 问题描述
- 只标记确实值得关注的特征（无鉴权、参数直接流入危险操作、URL 拼接等）

### 分析建议

- 推荐方向：source / sink / 两者
- 优先级建议

### 格式要求

- 使用 Markdown 表格汇总暴露面
- 引用行号：`文件路径:行号`
- 代码片段使用代码块（` ``` `）
- 可疑点使用项目符号加粗标注

## 质量门槛

一个完整的暴露面映射必须包含：
- 至少识别了项目语言和框架
- 至少覆盖了 HTTP 路由分类（大多数项目都有）
- 对每个分类，覆盖了该框架下所有可能的入口注册方式
- 内部调用已被正确排除
- 至少标记了 1 个可疑点（如有）
- 每个暴露面引用了具体文件:行号
- 无模糊用语（"可能"、"大概"）——不确定时使用"不明确"

## 格式一致性

- 分类名全大写：HTTP / RPC / MQ / CLI / CRON / FILE_IO / OUTBOUND_HTTP
- 位置格式：`相对路径/文件名:行号`
- 多条发现用项目符号
- 分类间用 `---` 分隔

# 文件风险分析 (File Risk Analyzer)

你收到一批文件路径，需要逐一进行风险分析并定级。每个文件独立分析，互不干扰。

## 输出目录

分析结果写入当前目录下的 `.vuln_agent_output/file_rksk/` 中，按源文件的全路径生成子目录结构：

```
源文件: /opt/myproject/Main.java
输出:   .vuln_agent_output/file_rksk/opt/myproject/Main.java.md
```

**路径解析规则：**
- 若收到的是相对路径，基于当前工作目录将其解析为绝对路径
- 从绝对路径去掉前导 `/`，再追加 `.md` 作为输出文件路径
- 输出根目录为当前目录下的 `.vuln_agent_output/file_rksk/`
- 确保输出文件的所有父目录存在（不存在则创建）
- 输出文件已存在则直接覆盖

## 分析流程

### Step 0：解析文件列表，准备输出路径

遍历每个输入文件：
1. 将其解析为绝对路径（结合当前工作目录）
2. 构造输出路径：`{当前目录}/.vuln_agent_output/file_rksk/{去掉前导/的绝对路径}.md`
3. 创建该输出文件所需的所有父目录

### 对每个文件循环执行 Step 1 ~ Step 4

#### Step 1：一次性读取

一次性读取该文件全文。读取完成后不再对文件做任何关键词搜索或 Grep 匹配——后续分析完全依赖本次读取的内容进行语义理解。

在审查风险前，基于文件内容识别一句话的业务上下文（如"CSV 数据处理"、"Elasticsearch 查询"、"ZIP 解压上传"、"用户注册登录"），用于输出时标注每个风险点所处的业务环节。

#### Step 2：逐段审查七类风险

基于已读入的文件内容，逐段审查以下七类风险。禁止对文件做任何关键词搜索或 Grep 匹配——必须依赖 Step 1 读入的内容进行语义分析。

**关键原则：** 对每个疑似风险点，必须追溯所有涉及的变量来源，确认是外部可控还是内部硬编码。**"外部"指当前文件之外**——函数参数、全局变量、import 的模块变量、网络/DB 读取的值、环境变量、用户输入等都属于外部变量。文件内部字面量/常量定义属于内部变量。注意：**⑤ 端口监听不论变量来源均为高风险。**

#### ① 命令/代码执行
检查是否存在调用系统命令、启动子进程、动态执行代码的逻辑（如系统命令调用、子进程创建、eval/exec 类动态执行、表达式求值等）。

#### ② 路径类文件操作
检查文件读写、移动、删除、上传、下载等操作，关注路径参数是否由外部数据拼接而成。

#### ③ SQL 拼接
检查数据库查询语句中是否混入了字符串拼接或插值操作。

#### ④ 网络请求
检查 HTTP 请求或网络请求的 URL 是否包含拼接的外部数据（包括 URL 拼接、HTTP 请求发送、REST API 调用、Socket 客户端连接、gRPC 调用等）。

#### ⑤ 网络服务/端口监听
检查是否存在端口绑定、HTTP/FTP/SSH 等服务监听、Socket 服务端 accept() 等行为。**此项不区分变量来源，统一为高风险。**

#### ⑥ 直接安全漏洞
- **证书校验绕过：** 证书验证被跳过或设为不校验
- **硬编码认证凭据：** 密码、密钥、令牌等认证凭据以字面量形式出现在代码中
- **ReDoS：** 正则模式中存在嵌套量词等可导致 catastrophic backtracking 的结构

#### ⑦ 敏感信息日志
检查日志输出或打印语句中是否输出了疑似密码、密钥、令牌、证书、会话标识等敏感变量。

#### Step 3：定级

对每个确认的风险点，按以下规则定级：

| 类别 | 具体条件 | 评级 |
|---|---|---|
| 命令/代码执行 | 外部变量注入，且实际执行 | **高** |
| 命令/代码执行 | 变量均来自文件内部，且实际执行 | **中** |
| 命令/代码执行 | 仅定义字符串/模板，无调用 | **信息** |
| 路径类文件操作 | 路径由外部变量拼接，且实际操作 | **高** |
| 路径类文件操作 | 路径拼接但变量内部，且实际操作 | **中** |
| 路径类文件操作 | 仅拼接路径字符串，无实际操作 | **信息** |
| SQL 拼接 | 查询混入外部变量拼接/插值 | **高** |
| SQL 拼接 | 查询拼接但变量内部 | **中** |
| SQL 拼接 | 仅定义 SQL 模板，无数据库调用 | **信息** |
| 网络请求 | URL/请求参数由外部变量拼接 | **高** |
| 网络请求 | 拼接但变量内部 | **中** |
| 网络请求 | 仅构建请求对象，无实际发送 | **信息** |
| 网络服务/端口监听 | 存在端口监听/服务绑定行为 | **高** |
| 证书校验绕过 | 证书验证被跳过或设为不校验 | **高** |
| 硬编码凭据 | 密码/密钥/令牌以字面量出现 | **高** |
| ReDoS | 正则含嵌套量词等可致 catastrophic backtracking | **高** |
| 敏感信息日志 | 日志输出来自外部的疑似敏感变量 | **高** |
| 敏感信息日志 | 仅凭变量名推测含敏感信息（值在文件内部） | **中** |

**优先级规则：** 一个代码片段如果同时命中多条规则，取最高评级。

#### Step 4：写入文件

将分析结果写入输出文件。无风险则写入"**该文件未发现风险点。**"

每条风险点使用以下格式：

```
**N. 风险类型 · 行号 X-Y**
**风险评级：** 高/中/信息
**业务上下文：** 一句话描述
**代码片段：**
```language
code here
```
**数据来源：** `变量名` ← 追溯链（外部直连 / 外部间接 / 内部）
```

**字段说明：**
- **风险类型：** 命令注入 / 代码注入 / 路径遍历 / SQL 注入 / 网络请求注入 / 端口监听 / 证书校验绕过 / 硬编码凭据 / ReDoS / 敏感信息泄漏
- **行号：** 风险代码的起始行-结束行
- **评级：** 高 / 中 / 信息
- **业务上下文：** 一句话描述该文件在系统中所处的业务环节（如"CSV数据处理"、"Elasticsearch查询"、"ZIP解压上传"、"用户注册登录"等），基于文件内容自然推断
- **代码片段：** 关键代码块（无需整段，突出风险部分即可）
- **数据来源：** 标明每个外部变量的来源类型：
  - `外部直连` — 直接来自用户输入/网络请求/文件上传等（最危险）
  - `外部间接` — 来自函数参数、全局变量、import 模块、环境变量、数据库读取等
  - `内部` — 文件内硬编码字面量或常量

**示例：**
```
**1. 命令注入 · 行号 85-90**
**风险评级：** 高
**业务上下文：** 视频上传处理
**代码片段：**
```python
cmd = f"ffmpeg -i {video_path} -vcodec libx265 output.mp4"
os.system(cmd)
```
**数据来源：** `video_path` ← `request.files['video'].filename`（外部直连）
```

写入完成后，在 stdout 输出确认：`分析结果已写入: {输出路径}`

（继续处理下一个文件，直到所有文件分析完毕）

## 纪律

- **不遗漏：** 七类风险都要扫一遍，不要只看命中的就停
- **不猜测：** 追溯变量来源必须以代码行级证据为准，不能凭文件名猜测
- **不重复：** 同一段代码命中多个风险类型时合并为一条输出，评级取最高
- **不输出无关信息：** 只输出风险点列表，不需要额外说明、总结或建议
